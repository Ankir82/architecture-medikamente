| Код | Требования                         | Комментарий  |
|:---:|:----------------------------------------------------------------------------------|:----------------------------------|
| F   | **Функциональные (Functionality)**     |              |
| F1  | На сайте банка отображается список доступных  депозитов с актуальными ставками  |  Только для новых клиентов  |
| F2  | Клиент может подать заявку на депозит, заполнив поля номер телефона и ФИО  |  Заявка должна автоматически попадать в систему кол-центра |
| F3  | Менеджер кол-центра видит список заявок в системе кол-центра и может предложить особые условия клиенту |              |
| F4  | В интернет банке отображается список доступных депозитов с актуальными ставками  |              |
| F5  | В интернет банке отображается список персонализированных ставок лично для клиента |              |
| F6  | Клиент может указав счет и сумму депозита подать заявку на его открытие |              |
| F7  | Клиент должен иметь возможность подтвердить заявку на депозит с помощью СМС-кода |              |
| F8  | АБС система должны вестить актуальные ставки по депозитам | необходимо реализовать как просмотр, так и редактирование ставок |
| F9  | Менеджер back-оффиса может подтвердить заявку на депозит в АБС | Актуально для MVP |
| U   | **Удобство использования (Usability)** |              |
| U1  | Все новые интерфейсы соответствуют корпоративной дизайн системе |              |
| U2  | Для ключевых страниц системы время отклика должна быть < 1 сек  |              |
| R   | **Надёжность (Reliability)**       |              |
| R1  | Доступность сервисов >=99,9%         |              |
| R2  | Время работы сервисов 24/7         |              |
| R3  | Должна быть реализована отказаустойчивость система на базе 2 ЦОДов |              |
| R4  | Интеграция с АБС на запись должна быть ассинхронной  и не терятся пр сбоях интеграции | Предполагается использование kafka |
| P   | **Производительность (Performance)** |              |
| P1  | Время загрузки ключевых страниц системы должно быть < 1 сек | Предполагается использование Redis для кеширования и частично дублирование справочников АБС и обновлением через паттерн издатель-подписчик |
| P2  | Возможность горизонтального масштабирования всех новых компонентов системы |              |
| S   | **Поддерживаемость (Supportability)** |              |
| S1  | При разработке используются уже имеющиеся в банке стеки технологий | .NET, базы данных MS SQL и Oracle, брокер сообщений kafka |
| S2  | Разработка документации для дальнейшего расширения системы |              |
| S3  | Должна быть поддержка интеграционного тестирования и настроены CI/CD | Легче расширять функциональность |
| +R  | **+ Ограничения (Restrictions)** |              |
| +R1 | АБС масштабируется только вертикально из-за своей базы данных |              |
| +R2 | Весь клиентский и межсистемный трафик должен шифроваться |              |
| +R3 | в случае сбоев других систем неообходимо, чтобы сервисы интернет-банка были доступны и выдерживали требуемую нагрузку                                |              |
